## Project Overview

SpaceWorm is a retro pixel-art UI theme for Godot 4 (targeting 4.7, Forward Plus renderer), distributed as public domain on itch.io. It provides styled versions of all standard Godot UI controls with a space/retro aesthetic in blue and sand color variants.

## Running / Editing the Theme

There is no CLI build system. Open `space-worm-theme-workshop/` as a Godot 4.7 project in the Godot editor. The main scene (`Scenes/themeTester.tscn`) is a live preview of all themed controls — run it to see theme changes in action.

To export: use **Project > Export** in the Godot editor with the configured presets in `export_presets.cfg`.

## Architecture

The repository has two parallel directories that stay in sync:

- **`space-worm-theme-workshop/`** — the Godot project used for development and preview. Theme assets live at `Themes/space-worm-theme/`.
- **`dist/space-worm-theme/`** — the standalone distributable drop-in, a copy of the theme assets without the workshop project. When assets change in the workshop, they must also be updated here.

### Theme resource

The single source of truth for all styling is:
`space-worm-theme-workshop/Themes/space-worm-theme/space_worm_theme.tres`

This is a Godot `Theme` resource (text format) that wires PNG `StyleBoxTexture`s and `StyleBoxFlat`s to every UI control state. It should be edited through the Godot Theme editor, not manually — the file is auto-generated by the editor.

### Asset layout under `Themes/space-worm-theme/`

| Folder | Contents |
|---|---|
| `button/` | Button state images (normal Blue, hover, close variants) |
| `checkbox/` | CheckBox and radio icons — Blue and Sand color variants |
| `fonts/` | ProggyClean (default), CozetteVector, PixelCode (5 weights), PixeloidMono, DepartureMono |
| `general/` | Shared nine-patch textures: Blue, LighterBlue, Green, Purple, arrows, spinner, panel outline |
| `icons/` | SpaceWorm branding icons |
| `panel/` | Panel and Tab backgrounds |
| `scrollbar/` | Scroll/slider tracks and grabbers |
| `separator/` | Separator line images |
| `textedit/` | TextEdit/LineEdit backgrounds (White, Black, Disabled) |
| `space-worm-colors.clrs` | Godot color palette file |

### Color variants

Several assets exist in two color families: **Blue** (primary) and **Sand** (secondary). The active theme uses Blue variants; Sand assets are available for customization.

### Workshop scripts

- `Scripts/ThemeTesterScript.gd` — handles the About/Quit menu bar items in the preview scene
- `Scenes/tree.gd` — populates the Tree widget in the preview scene

There is intentionally very little GDScript; this project is about the visual theme, not game logic.

## Workflow for Theme Changes

1. Open `space-worm-theme-workshop/` in Godot 4.7
2. Edit assets (PNG files in the theme subfolders) with a pixel art editor (Aseprite-style assets are in `Inspo/`)
3. Edit `space_worm_theme.tres` via the Godot Theme editor to update `StyleBoxTexture` margins/regions as needed
4. Run the main scene to verify all controls look correct
5. Copy changed assets and the `.tres` file to the corresponding paths under `dist/space-worm-theme/`
